## 缓存策略

### Network or cache

#### 要求：
    尽快的显示数据

#### 问题：
    发起请求但是请求时间过长，需要从缓存中读取数据

#### 解决：
    包含一个超时，超时返回缓存数据

#### 总结：
    1. 每次在 cache 中更新某个数据（使用 delete 再 add 或 put 操作，重新载入资源），这个时候 serviceWorker 都会发起一次真正的网络请求，以缓存对应的资源。如果是一个每次请求都返回不同数据的资源，不可使用该策略。

    2. 熟悉 WebAPI fetchEvent 事件

    3. 熟悉 WebAPI cache
